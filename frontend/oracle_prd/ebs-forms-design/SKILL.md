# EBS 功能设计文档 → 可交互原型

从一份 Oracle EBS 功能设计文档（Markdown 格式）生成一个完整的可交互原型 HTML 文件。

## 触发条件

当用户提出"生成可交互原型"、"做原型"、"做HTML"等请求，且内容涉及 Oracle EBS 功能设计时，加载本 skill。

## 前置引导

**不要直接问"文档在哪里"就跳过引导。** 必须先执行以下引导流程，引导用户逐步提供信息。

### Step 1 — 询问功能设计文档

首先询问用户是否有现成的功能设计文档（`.md` 格式），如果有则直接读取。

### Step 2 — 无文档时的信息收集

如果用户没有现成文档，依次（分步）询问以下内容：

1. **功能背景介绍** — 什么业务场景？为什么要做这个功能？原系统是什么？
2. **方案介绍** — 整体解决方案思路，主要变化点
3. **总体功能架构** — 分为哪几个功能模块？每个模块的核心作用
4. **功能需求点** — 每个功能模块下的具体需求：界面字段、操作按钮、业务逻辑规则、处理流程

每次只问一个问题，用户回答后再问下一个，不要一次性全抛给用户。

### Step 3 — 最低保底信息

如果以上信息用户都无法提供，至少问清：**"您想做什么功能？"** 描述清楚核心业务流程。基于这个描述，由你来梳理和补充功能架构。

### Step 4 — 确认导出位置

告知原型导出目录默认为项目内 `frontend/oracle_prd/exports/`，并说明系统会按一次导出生成一个独立目录（包含 `index.html`、样式、脚本及压缩包）。如用户明确要求其它位置，再单独确认。

### Step 5 — 示例参考

如果用户想了解最终效果，可打开示例文件 `examples/供应商对账功能_原型.html` 供用户预览。打开方式：使用 `read_file` 查看文件内容，或使用 `open` 命令在浏览器中打开。

示例文件展示了：
- 左侧树形导航 + 右侧功能面板布局
- 顶部全局工具栏（截图、批量截图、生成功能文档、缩放、聚焦、全屏、帮助）
- 折叠/展开工具栏
- 每个按钮点击弹出业务说明弹窗
- 生成功能设计文档（Word .docx / Markdown .zip）
- 聚焦模式（点击任意区域或按 ESC 退出）

## 步骤

### 0. 确定布局模式

根据需求规模和风格，在开始前选择一种布局模式，并告知用户选择。

#### 模式A — 窗口式（带全局标题栏）

适用于 7 个以上功能模块的正式原型，模拟 EBS Forms 完整窗口。

```
┌─ 全局标题栏 ───────────────┬─ □ × ─┐
├──────┬─────────────────────────────┤
│ 导航  │  ┌─ 面板标题栏 ─┬─ □ × ┐ │
│ 栏    │  ├─ 面板内容 ────┤      │ │
│ 210px │  ├─ 面板状态栏 ──┤      │ │
│       │  └────────────────┘      │
├──────┴─────────────────────────────┤
│ 全局状态栏                         │
└────────────────────────────────────┘
```

#### 模式B — 独立式（无全局标题栏）

适用于简洁嵌入式原型，一打开就是导航+面板，四周留白。

```
┌─────────────────────────────────────┐
│  (body padding: 30px)               │
│  ┌──────┬────────────────────────┐  │
│  │ 导航  │  面板标题栏 ─┬─ □ ×   │  │
│  │ 标题  │  ───────────┤         │  │
│  │ 210px │  面板内容              │  │
│  │       │  面板状态栏            │  │
│  └──────┴────────────────────────┘  │
└─────────────────────────────────────┘
```

#### 模式C — 按钮交互模式

**C1. 描述模式**：每个按钮点击弹出说明弹窗，含"📌 业务逻辑"+"🖱️ 交互逻辑"。
**C2. 动作模式**：按钮直接执行模拟操作（查询、生成、发送等）。
默认 C2，可在同一原型中混用。

### 1. 理解文档/需求

完整理解以下内容：
- 业务背景、需求清单/开发项、流程设计
- 表单/界面设计、数据项设计、程序逻辑说明、凭证字段映射表

### 2. 设计导航结构

分析功能模块的层次关系，设计左侧树形菜单。

### 3. 生成原型 HTML

生成可预览的原型 HTML，并按当前项目实现导出到 `frontend/oracle_prd/exports/{导出目录名}/`。默认应与现有导出机制保持一致，不要声称已写入 `~/Downloads/` 或其它未验证路径。

详见同目录 `references/` 中的设计规范和功能实现文档。

### 4. 验证

确保导出目录中的 HTML 在浏览器中可直接打开使用，所有交互正常运行；回复用户前先核对导出目录或文件确实存在。

## 注意事项

- **单文件**：所有 CSS/JS/HTML 位于同一个 `.html` 文件中
- **字体** `宋体, SimSun, SimSun-ExtB, serif`
- **字号规范**：

| 元素 | 字号 | 说明 |
|------|------|------|
| 页面正文（html 基准） | **10pt** | 标题栏、表单标签、输入框、section 标题等全部正文 |
| 导航栏（标题/文件夹/叶子） | **14px** | 标题 #2D6EB0 白底，节点选中 #2D82C3 白字 |
| 按钮（含工具栏按钮） | **10pt** | 查询、新建、保存、截图等所有操作按钮 |
| 表格单元格（td） | **10pt** | data-table、scroll-table、std-table |
| 表格表头（th） | **10pt** | 渐变背景 + outset 边框 |
| 数据项表（data-items） | **10pt** | 字段设计明细表（含 th/td） |
| 状态标签 `.tag` | **9pt** | 待处理、已完成、例外等标签 |
| 窗口控制按钮 | **10px** | 最小化/最大化/关闭 |
| 状态栏 | **9pt** | 底部信息栏 |
- **布局模式** 在 Step 0 中与用户确认
- **导航对齐** 模式B 中 `view-panel { margin: 0 6px }`
- **四周留白** body 用 `display: flex; justify-content: center; align-items: center; padding: 30px;`
- **窗口阴影** `.window` 用 `box-shadow: 4px 4px 12px rgba(0,0,0,0.25)`
- **全局工具栏**（必需）每个原型必须添加
- **标题栏 Oracle 图标（必加）** 每个功能面板标题栏左侧必须显示 Oracle favicon。使用 skill 资产 `assets/orcale_favicon.png` 的 base64 data URI 内嵌到 CSS 中（类名 `.view-title-icon`），图标尺寸 20×20px。**禁止**使用 `https://www.oracle.com/favicon.ico` 外部 URL — html2canvas 截图时因 CORS 限制图标空白。详见 `references/screenshot-feature.md` 的"图标坑点"章节。
- **浏览器 tab favicon** 同时添加 `<link rel="icon">` 到 `<head>`，使用同一 base64 data URI。
- **截图文字裁剪** 输入框加 `line-height: 20px; vertical-align: middle; box-sizing: border-box;`
- **截图与文档生成** 见 `references/screenshot-feature.md`
- **聚焦模式陷阱** 不要使用 `.focus-overlay` 的 `onclick` 退出，因为 `.view-panel` 的 z-index(10000) 高于遮罩层(9998)，点击面板时点不到遮罩。改为全局 `document.addEventListener('click')` + 检查 `body` 是否含 `data-focus-exit` 属性。注意排除聚焦按钮自身的点击——按钮点击后设置 `data-focus-exit`，click 冒泡到 document 会立即退出。用 `e.target.closest('.tb-btn')` 检查 onclick 是否包含 `toggleFocus`，是则跳过。
- **聚焦模式退出方式** 点击面板外部（背景/遮罩）退出 + 按 ESC 键退出。点击面板内部按钮/表格/输入框时不退出。实现：`if (focusMode && e.target.closest && e.target.closest('.view-panel')) return;`
- **状态栏截图干净** 状态栏左侧（`.status-left`）HTML 中设为空字符串，`showMsg()` 仅更新 toast（`#msgToast`），不写入 status-left。截图时无需临时清空，始终干净。
- **showMsg 职责分离** `showMsg()` 只做两件事：更新 toast 显示并 2 秒后隐藏、更新 `#statusMsg` 置空（或删除对 status-left 的赋值）。消息反馈仅通过底部浮层，不污染状态栏文字。
